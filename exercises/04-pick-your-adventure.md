# Block 4: Pick Your Adventure

**Time:** 35 minutes

## Goal

Choose 1–2 exercises that match what YOU need right now. This is your time to explore the PM skills collection and walk away with something you'll use on Monday.

Browse the menu below and pick what resonates.

> **Not sure where to start?** Look for the ⭐ — those are recommended for first-timers.

---

## For Product Owners

### A. Competitive Analysis
*Plugin: pm-market-research*

```
Run a competitive analysis for my product.

Identify 3-5 competitors (or use these: [list your known competitors]).
For each, analyze: strengths, weaknesses, positioning, pricing, and target audience.
Then show where we have differentiation opportunities.

Save to deliverables/competitive-analysis.md
```

### B. Write a PRD ⭐
*Plugin: pm-execution — great if you have a feature idea you've been meaning to write up*

```
Write a PRD for this feature idea: [describe a feature you're considering]

Use my product context. Include:
- Problem statement and user need
- Proposed solution
- Success metrics
- Scope (in/out)
- Open questions and risks

Save to deliverables/prd-[feature-name].md
```

### C. OKRs for Next Quarter ⭐
*Plugin: pm-execution — useful if your team is approaching quarterly planning*

```
Using my product context and strategy, brainstorm OKRs for next quarter.

Create 2-3 Objectives with 3-4 Key Results each.
Align them with the product strategy and current team capacity.
Flag which KRs are stretch vs. committed.

Save to deliverables/okrs-next-quarter.md
```

### D. North Star Metric
*Plugin: pm-marketing-growth*

```
Help me define a North Star Metric for my product.

Classify my business game (attention, transaction, or productivity).
Propose a North Star Metric and 4-5 input metrics that drive it.
Explain how each input metric connects to the North Star.

Save to deliverables/north-star-metric.md
```

### E. User Personas & Journey Map
*Plugin: pm-market-research*

```
Using my product context, create:
1. 2-3 user personas with demographics, goals, frustrations, and JTBD
2. A customer journey map for the primary persona (stages, touchpoints, emotions, pain points)

Save to deliverables/personas-and-journey.md
```

## For Scrum Masters

### F. Stakeholder Map ⭐
*Plugin: pm-execution — especially useful if you're navigating complex stakeholder dynamics*

```
Create a stakeholder map for my product using the Power × Interest grid.

Based on my team and product context, identify key stakeholders.
For each: classify their power and interest level, and suggest a communication approach.
Create a communication plan (who gets what, how often, in what format).

Save to deliverables/stakeholder-map.md
```

### G. Sprint Planning Assistant
*Plugin: pm-execution*

```
Help me plan the next sprint.

Using my product context and backlog, suggest:
1. A sprint goal aligned with our current product goal
2. Which stories to pull in (based on priority and team capacity)
3. Capacity estimation and risk flags
4. Dependencies to watch out for

Save to deliverables/sprint-plan.md
```

### H. Pre-Mortem ⭐
*Plugin: pm-execution — great for upcoming releases or risky initiatives*

```
Run a pre-mortem on our current product goal / upcoming release.

Imagine it's 3 months from now and things went wrong. Classify risks as:
- Tigers (real, high-impact threats)
- Paper Tigers (feel scary but manageable)
- Elephants (big issues everyone avoids talking about)

For each risk, suggest a mitigation action.

Save to deliverables/pre-mortem.md
```

### I. Meeting Notes Summarizer
*Plugin: pm-execution*

```
Here are the notes from a recent meeting:

[Paste your meeting notes or transcript]

Summarize into:
1. Key decisions made
2. Action items (who, what, by when)
3. Open questions
4. Parking lot items

Save to deliverables/meeting-summary.md
```

## For Everyone

### J. Analyze Feature Requests
*Plugin: pm-product-discovery*

```
Here are feature requests from our users/customers:

[Paste feature requests, support tickets, or feedback]

Analyze and categorize them by:
1. Theme / cluster
2. Strategic fit (aligned with strategy or not)
3. Frequency and urgency
4. Recommended action (build, investigate, park, decline)

Save to deliverables/feature-request-analysis.md
```

### K. Launch Planning
*Plugin: pm-go-to-market*

```
Create a go-to-market plan for [describe what you're launching].

Include:
1. Beachhead segment — who do we target first?
2. Messaging — key value props for this audience
3. Channels — how do we reach them?
4. Success metrics — how do we know it worked?
5. Timeline — key milestones

Save to deliverables/launch-plan.md
```
