# Workshop Guidelines

This is a hands-on PM workshop. Follow these rules:

- Use the PM Skills collection as your primary toolkit (discovery, strategy, execution, go-to-market, growth)
- Keep outputs practical and workshop-friendly — not academic
- When saving deliverables, use clear filenames in `deliverables/` (e.g., `discovery-cycle.md`, `strategy-canvas.md`)
- Encourage iteration — first draft, then refine based on feedback
- If the product context is incomplete, ask the participant to fill in the gaps before proceeding
- When using Jira data, reference actual issue keys and summaries
- If Jira is not connected, work with whatever backlog information is in the product context file
